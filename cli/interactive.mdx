title: 'hyve interactive'
description: 'Guided terminal user interface for all Hyve commands'
---

The `hyve interactive` command (alias: `hyve tui`) launches a full-screen terminal user interface that lets you navigate and execute any Hyve operation without memorising flags or command syntax.

## Usage

```bash
hyve interactive
hyve tui          # alias
```

No flags or arguments are required. The TUI takes over the terminal and guides you step by step.

---

## Navigation

| Key | Action |
|-----|--------|
| `↑` / `↓` | Move between options |
| `Enter` | Select / confirm |
| `Shift+Tab` | Go back within a multi-step form |
| `← Back` option | Return to the previous menu |
| `Quit` option | Exit the TUI from the root menu |
| `Ctrl+C` | Exit at any point |

The TUI is fully loop-based — after completing an action you are returned to the same menu, so you can run multiple operations in a session without re-launching.

---

## Root Menu

```
Hyve — what would you like to do?
  > cluster    — manage Kubernetes clusters
    git        — manage Git repositories
    config     — provider credentials & config
    workflow   — automated pipelines
    template   — reusable cluster patterns
    kubeconfig — cluster access
    Quit
```

Selecting any section drills down into a sub-menu for that command group. Selecting **← Back** from any sub-menu returns you to the root.

## Sections

### cluster

| Action | Description |
|--------|-------------|
| Add a new cluster | Provider, region, node sizes, and account aliases collected via guided prompts. Account/VPC/role fields are populated from your configured aliases. |
| Modify an existing cluster | Cluster selected from a live list. |
| Delete a cluster | Cluster selected from list; choice between GitOps (state-only) and immediate cloud deletion. |
| Force-delete from cloud | Cluster and provider details collected; destructive action requires explicit confirmation. |
| List clusters | Prints all cluster definitions. |

### git

**Repository management**

| Action | Description |
|--------|-------------|
| Add repository | URL, alias, and optional username collected via form. |
| List repositories | Prints all configured repos. |
| Use (switch to) repository | Repository selected from list. |
| Show status | Displays current repo status. |
| Remove repository | Repository selected from list; requires confirmation. |

**Branch management**

| Action | Description |
|--------|-------------|
| List branches | Prints all branches. |
| Create branch | Name, switch-after, and push options collected via form. |
| Switch branch | Branch name and optional pull. |
| Delete branch | Requires confirmation; safe or force delete. |

**Pull / push / sync**

| Action | Description |
|--------|-------------|
| Pull | Pulls latest changes from remote. |
| Push | Commit message collected then changes pushed. |
| Sync | Commit message collected; pulls then pushes. |

### config

**Civo** — set, get, clear, and list tokens. Org aliases populated from configured list.

**GCP** — add, list, get, and remove project aliases.

**AWS** — three sub-sections:
- **Account aliases** — add, list, get, remove.
- **EKS role aliases** — add (register ARN), create (provisions in AWS), list, get, remove. Account selected from list before role.
- **VPC aliases** — add, create, list, get, remove. Account selected from list before VPC.

**Azure** — two sub-sections:
- **Subscription aliases** — add, list, remove.
- **Resource groups** — add (creates in Azure), list, delete. Subscription selected from list before resource group.

### workflow

| Action | Description |
|--------|-------------|
| Create | Choose default template or import an existing YAML file. |
| Run | Workflow selected from list; optional cluster and log/output flags. |
| List | Prints all workflows. |
| Show | Workflow selected from list; displays full definition. |
| Validate | Workflow selected from list; reports errors and warnings. |
| Delete | Workflow selected from list; requires confirmation. |

### template

| Action | Description |
|--------|-------------|
| Create | Provider, region, node sizes, and cluster type collected via form. |
| Execute | Template selected from list; cluster name and provider-specific aliases collected. Account/VPC/role fields populated from configured aliases. |
| List | Prints all templates. |
| Show | Template selected from list. |
| Validate | Template selected from list. |
| Delete | Template selected from list; requires confirmation. |

### kubeconfig

| Action | Description |
|--------|-------------|
| Sync | Retrieves kubeconfigs from all active clusters. |
| Get | Cluster selected from stored kubeconfigs. |
| Use | Cluster selected; merges into `~/.kube/config` and sets active context. |
| Merge | Cluster selected; merges into `~/.kube/config`. |
| Remove | Cluster selected from list; requires confirmation. |

## Smart Selects

Wherever possible, the TUI populates select menus from live data rather than requiring typed input:

| Field | Source |
|-------|--------|
| Cluster name | State YAML files in current repository |
| Workflow name | `workflows/` directory in current repository |
| Template name | `templates/` directory in current repository |
| Git repository | Local repository database |
| Kubeconfig cluster | Stored kubeconfig database |
| AWS account alias | `provider-configs/aws.yaml` |
| AWS EKS role alias | `provider-configs/aws.yaml` (scoped to selected account) |
| AWS VPC alias | `provider-configs/aws.yaml` (scoped to selected account) |
| GCP project alias | `provider-configs/gcp.yaml` |
| Azure subscription alias | `provider-configs/azure.yaml` |
| Azure resource group | `provider-configs/azure.yaml` (scoped to selected subscription) |
| Civo org alias | `provider-configs/civo.yaml` |

If no items are configured for a field (e.g. no AWS accounts added yet), the TUI falls back to a free-text input automatically.

---

## Color Scheme

The TUI uses a yellow and blue theme:

- **Yellow** (`#F5C518`) — titles, selected option indicators, text input cursor
- **Blue** (`#4A9FD5`) — field borders, text input prompt
- **Dark blue** (`#1E6FA8`) — focused button background
