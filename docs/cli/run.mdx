---
title: 'hyve run'
description: 'Execute a command with the kubeconfig scoped to a specific cluster'
---

`hyve run` executes an arbitrary command with `KUBECONFIG` set to a specific cluster's credentials — without permanently changing your active `kubectl` context. Useful for one-off operations against a cluster or for scripting multi-cluster workflows.

## Usage

```bash
# Individual arguments mode
hyve run --cluster <cluster> <command> [args...]

# String mode (supports flags and pipes)
hyve run --cluster <cluster> --string "<command string>"
```

## Flags

<ParamField path="--cluster, -c" type="string">
  Cluster name to use for the kubeconfig. If omitted and only one cluster is available, it is selected automatically. If multiple clusters exist, the flag is required.
</ParamField>

<ParamField path="--string, -s" type="string">
  Run a command from a string rather than individual arguments. Supports flags, quoted values, and basic shell syntax (pipes are not executed by the shell — use `--string` with `sh -c` for piped commands). Cannot be combined with positional command arguments.
</ParamField>

## Examples

### Run kubectl commands

```bash
# List nodes on the production cluster
hyve run --cluster production kubectl get nodes

# Get pods with additional flags
hyve run --cluster staging kubectl get pods -n default -o wide

# Describe a deployment
hyve run --cluster dev kubectl describe deployment my-app -n default
```

### Use string mode for commands with flags

```bash
# String mode avoids flag parsing ambiguity
hyve run --cluster production --string "kubectl -n=kube-system get pods"

# Run helm commands
hyve run --cluster staging --string "helm list -A"
```

### Piped commands (via shell)

```bash
# Use sh -c for piped commands
hyve run --cluster production --string "sh -c 'kubectl get pods | grep nginx'"
```

### Auto-select single cluster

```bash
# If only one cluster is configured, --cluster can be omitted
hyve run kubectl get nodes
```

## How It Works

1. Looks up the kubeconfig for the specified cluster from local storage
2. Writes a temporary kubeconfig file to `~/.hyve/temp/`
3. Sets `KUBECONFIG` to the temporary file for the duration of the command
4. Executes the command, connecting its stdin/stdout/stderr to the terminal
5. Deletes the temporary file on exit

The active `kubectl` context in `~/.kube/config` is **not modified**.

<Info>
  Run `hyve kubeconfig sync` first to ensure the cluster's kubeconfig is stored locally.
</Info>

<Warning>
  `--string` and positional command arguments are mutually exclusive. Using both will result in an error.
</Warning>
