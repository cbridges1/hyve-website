The `hyve kubeconfig` command group provides utilities for managing your local `~/.kube/config`. Cluster authentication — fetching and merging a cluster's kubeconfig — is handled by `hyve cluster auth`, which runs the module's auth operation directly.

## Commands

### hyve kubeconfig remove

Remove a cluster context entry from `~/.kube/config`. Use this after a cluster has been deleted to keep your local kubeconfig tidy.

```bash
hyve kubeconfig remove <cluster-name>
```

<ParamField path="cluster-name" type="string" required>
  Name of the cluster context to remove
</ParamField>

**Example:**

```bash
hyve kubeconfig remove my-cluster
```

**Output:**

```
📦 Backup created at /Users/username/.kube/config.backup
🗑️  Removing cluster 'my-cluster' from /Users/username/.kube/config
✅ Successfully removed cluster 'my-cluster' from /Users/username/.kube/config

💡 View remaining contexts:
   kubectl config get-contexts
```

<Info>
  To add a cluster context, use `hyve cluster auth <name>` — it runs the module's auth operation (e.g. `civo kubernetes config --save`) and merges the result directly into `~/.kube/config`.
</Info>
