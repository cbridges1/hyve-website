## Overview

Templates in Hyve enable reusable cluster configurations with automated workflow execution. Create standardized cluster patterns once and deploy them consistently across environments.

## Template Basics

### What are Templates?

Templates combine:
- Cluster specifications (provider, region, nodes or node groups)
- Ingress configuration
- Lifecycle workflows (onCreated runs after cluster creation, onDestroy runs before deletion)
- Node group definitions for multi-pool AWS EKS, GCP GKE, and Azure AKS clusters
- Reusable patterns for consistent deployments

## Template Structure

### Complete Template Example

```yaml
apiVersion: v1
kind: Template
metadata:
  name: production-standard
  description: Standard production cluster with 3 large nodes, monitoring, and observability
  labels:
    environment: production
    tier: standard
spec:
  provider: civo
  region: NYC1
  nodes:
    - g4s.kube.large
    - g4s.kube.large
    - g4s.kube.large
  clusterType: k3s
  ingress:
    enabled: true
    loadBalancer: true
  workflows:
    onCreated:
      - setup-monitoring
      - configure-logging
      - deploy-observability
      - setup-backups
    onDestroy:
      - export-logs
      - final-backup
```

### Template Fields

#### Metadata

```yaml
metadata:
  name: my-template                # Required: Unique template name
  description: What it creates     # Optional: Human-readable description
  labels:                          # Optional: Key-value labels
    environment: production
    cost-center: engineering
```

#### Spec

```yaml
spec:
  provider: civo                   # Required: Cloud provider (civo, aws, gcp, azure)
  region: NYC1                     # Required: Deployment region
  nodes:                           # Optional: Simple node list (Civo single-pool clusters)
    - g4s.kube.large
    - g4s.kube.large
  nodeGroups:                      # Optional: Named node pools (AWS EKS, GCP GKE, Azure AKS)
    - name: workers                #   Pool name
      instanceType: t3.medium      #   Node instance/machine type
      count: 3                     #   Desired node count
      minCount: 1                  #   Minimum nodes (enables autoscaling)
      maxCount: 5                  #   Maximum nodes (enables autoscaling)
      diskSize: 50                 #   Root disk size in GB (optional)
      spot: false                  #   Use spot/preemptible instances (optional)
      mode: System                 #   Azure only: System or User pool (optional)
      labels:                      #   Node labels (optional)
        role: worker
      taints:                      #   Node taints (optional)
        - key: dedicated
          value: gpu
          effect: NoSchedule
  clusterType: k3s                 # Optional: k3s, talos, or eks (default: k3s)
  ingress:                         # Optional: Ingress configuration
    enabled: true
    loadBalancer: true
  workflows:                       # Optional: Lifecycle workflows
    onCreated:                     # Run after cluster is ready
      - workflow-name-1
      - workflow-name-2
    onDestroy:                     # Run before cluster deletion
      - cleanup-workflow

  # AWS-specific fields (only when provider: aws)
  awsAccount: main-account         # Account alias from provider-configs/aws.yaml
  awsVpcName: k8svpc               # VPC alias → resolved to VPC ID at execution time
  awsEksRole: k8srole              # EKS cluster role alias → resolved to role ARN
  awsNodeRole: k8snoderole         # EKS node role alias → resolved to role ARN
```

## Executing Templates

### Basic Execution

Execute a template to create a new cluster:

```bash
hyve template execute prod-template prod-cluster-01
```

**This command:**
1. Creates cluster definition from template
2. **For AWS**: resolves `awsVpcName`, `awsEksRole`, and `awsNodeRole` aliases to VPC IDs and role ARNs from `provider-configs/aws.yaml`
3. Provisions cluster in cloud provider
4. Waits for cluster to be ready
5. Syncs kubeconfig
6. Executes all `onCreated` workflows in order
7. Note: `onDestroy` workflows run when the cluster is deleted

### AWS EKS Template

EKS cluster using named node groups and aliases from your AWS provider config:

```yaml
apiVersion: v1
kind: Template
metadata:
  name: aws-eks-template
  description: Standard AWS EKS cluster with managed node groups
spec:
  provider: aws
  region: us-east-1
  nodeGroups:
    - name: system
      instanceType: t3.medium
      count: 2
      minCount: 1
      maxCount: 4
    - name: workers
      instanceType: t3.large
      count: 3
      minCount: 1
      maxCount: 10
      spot: true
      labels:
        role: worker
  clusterType: eks
  awsAccount: main-account    # matches accounts[*].name in provider-configs/aws.yaml
  awsVpcName: k8svpc          # matches accounts[*].vpcs[*].name
  awsEksRole: k8srole         # matches accounts[*].roles[*].name (EKS cluster role)
  awsNodeRole: k8snoderole    # matches accounts[*].roles[*].name (EKS node role)
  ingress:
    enabled: false
    loadBalancer: false
  workflows:
    onCreated:
      - deploy-app
```

<Info>
  AWS alias names (`awsAccount`, `awsVpcName`, `awsEksRole`, `awsNodeRole`) are resolved to actual IDs and ARNs from `provider-configs/aws.yaml` when `hyve template execute` runs. See the [AWS provider config](/configuration) for how to define these aliases.
</Info>

**Usage:**
```bash
hyve template execute aws-eks-template eks-prod-01
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Template Execution Fails">
    **Problem:** Template execution returns an error

    **Solutions:**
    ```bash
    # 1. Validate template first
    hyve template validate my-template

    # 2. Check API credentials
    hyve config list-tokens

    # 3. Verify workflows exist
    hyve workflow list

    # 4. Check region availability
    # Use valid regions: PHX1, NYC1, FRA1, LON1

    # 5. Review template YAML
    hyve template show my-template
    ```
  </Accordion>

  <Accordion title="AWS Alias Not Found">
    **Problem:** `hyve template execute` fails with "AWS VPC 'k8svpc' not found in account 'main-account'"

    **Cause:** The alias name in the template doesn't match any entry in `provider-configs/aws.yaml`.

    **Solution:** Verify your provider config contains matching alias names:
    ```yaml
    # provider-configs/aws.yaml
    accounts:
      - name: main-account         # must match awsAccount in template
        account_id: "123456789012"
        vpcs:
          - name: k8svpc           # must match awsVpcName in template
            vpc_id: vpc-0abc123
        roles:
          - name: k8srole          # must match awsEksRole in template
            role_arn: arn:aws:iam::123456789012:role/EKSClusterRole
          - name: k8snoderole      # must match awsNodeRole in template
            role_arn: arn:aws:iam::123456789012:role/EKSNodeRole
    ```
  </Accordion>

  <Accordion title="Workflows Fail During Execution">
    **Problem:** Template creates cluster but workflows fail

    **Solutions:**
    ```bash
    # Cluster is created but workflows failed
    # You can re-run workflows manually

    # 1. Sync kubeconfig
    hyve kubeconfig sync

    # 2. Run workflows individually
    hyve workflow run setup-monitoring --cluster my-cluster
    hyve workflow run deploy-app --cluster my-cluster

    # 3. Debug workflow issues
    hyve workflow validate setup-monitoring
    ```
  </Accordion>

  <Accordion title="Template Not Found">
    **Problem:** Template doesn't exist in repository

    **Solutions:**
    ```bash
    # List available templates
    hyve template list

    # Check template directory
    ls ~/.hyve/repositories/production/templates/

    # Sync from Git
    hyve git pull

    # Create template if missing
    hyve template create my-template --region PHX1 --nodes g4s.kube.medium
    ```
  </Accordion>

  <Accordion title="Cluster Already Exists">
    **Problem:** Cluster name already in use

    **Solutions:**
    ```bash
    # Use a different cluster name
    hyve template execute prod-template prod-cluster-02

    # Or delete existing cluster first
    hyve cluster delete prod-cluster-01
    hyve template execute prod-template prod-cluster-01
    ```
  </Accordion>

  <Accordion title="Invalid Node Sizes">
    **Problem:** Template validation warns about node sizes

    **Solution:**
    ```yaml
    # Use only valid Civo node sizes:
    nodes:
      - g4s.kube.xsmall   # 1 vCPU, 1 GB RAM
      - g4s.kube.small    # 1 vCPU, 2 GB RAM
      - g4s.kube.medium   # 2 vCPU, 4 GB RAM
      - g4s.kube.large    # 4 vCPU, 8 GB RAM
      - g4s.kube.xlarge   # 6 vCPU, 16 GB RAM
    ```
  </Accordion>
