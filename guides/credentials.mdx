## Overview

Hyve provides secure credential management with AES-GCM encryption for API tokens, Git credentials, and kubeconfigs. This guide covers storing, retrieving, and managing credentials securely.

## Security Architecture

### Encryption Details

Hyve uses industry-standard encryption:

- **Algorithm**: AES-256-GCM (Galois/Counter Mode)
- **Key Derivation**: PBKDF2 with machine-specific salt
- **Storage**: SQLite databases with encrypted values
- **In-Transit**: Never transmitted or logged unencrypted

### Storage Locations

```
~/.hyve/
├── credentials.db      # API tokens and Git credentials (encrypted)
├── kubeconfigs.db     # Cluster kubeconfigs (encrypted)
├── repositories.db    # Repository configurations (not encrypted)
└── temp/              # Temporary files (auto-cleaned)
```

<Warning>
  Never commit files from `~/.hyve/` to version control. The `.gitignore` should exclude this directory.
</Warning>

## API Token Management

### Supported Providers

Currently supported cloud providers:
- **Civo Cloud** - Required for cluster operations

### Storing API Tokens

<Tabs>
  <Tab title="Interactive (Recommended)">
    ```bash
    # Prompt for token (input hidden)
    hyve config set-token civo
    ```

    **Prompt:**
    ```
    Enter API token for civo (input will be hidden):
    ✅ API token for 'civo' stored successfully

    💡 The token is encrypted and stored in ~/.hyve/credentials.db
    💡 Hyve will now use this token automatically for civo operations
    ```
  </Tab>

  <Tab title="Command Line">
    ```bash
    # Provide token as flag (visible in history)
    hyve config set-token civo --token YOUR_TOKEN_HERE
    ```

    <Warning>
      Using `--token` flag exposes the token in shell history. Use interactive mode for better security.
    </Warning>
  </Tab>

  <Tab title="Environment Variable">
    ```bash
    # Set environment variable
    export CIVO_TOKEN=your_token_here

    # Add to shell profile for persistence
    echo 'export CIVO_TOKEN=your_token_here' >> ~/.bashrc
    source ~/.bashrc
    ```

    Priority: Stored token > Environment variable
  </Tab>
</Tabs>

### Retrieving API Tokens

```bash
# View stored token
hyve config get-token civo
```

**Output:**
```
🔑 API token for 'civo':
abc123def456ghi789jkl012mno345pqr678stu901vwx234yz
```

### Listing Stored Tokens

```bash
hyve config list-tokens
```

**Output:**
```
🔑 Stored API tokens (1):

  ✓ civo

💡 Commands:
  hyve config get-token <provider>    # View token
  hyve config clear-token <provider>  # Remove token
```

### Clearing API Tokens

```bash
# Remove token from storage
hyve config clear-token civo
```

**Output:**
```
✅ API token for 'civo' removed successfully
```

## Git Credential Management

### Storing Git Credentials

Store encrypted credentials for Git repository access:

<Tabs>
  <Tab title="Username and Password">
    ```bash
    # Store both username and password
    hyve git credentials --username your-username --password your_token
    ```

    **Output:**
    ```
    ✅ Credentials stored successfully

    Username: your-username
    Password: ********

    🔒 Credentials are encrypted and stored in ~/.hyve/credentials.db
    💡 These credentials will be used for all Git operations
    ```
  </Tab>

  <Tab title="Update Password Only">
    ```bash
    # Keep existing username, update password
    hyve git credentials --password new_token
    ```
  </Tab>

  <Tab title="Update Username Only">
    ```bash
    # Update username, keep existing password
    hyve git credentials --username new_username
    ```
  </Tab>
</Tabs>

<Info>
  For GitHub, use a Personal Access Token (PAT) as the password
</Info>

### Viewing Git Credentials

```bash
# Display current credentials
hyve git credentials
```

**Output:**
```
🔑 Git Credentials:

Username: your-username
Password: ********

🔒 Credentials are encrypted and stored in ~/.hyve/credentials.db
```

### Clearing Git Credentials

```bash
# Remove stored credentials
hyve git credentials --clear
```

**Output:**
```
✅ Git credentials cleared successfully
```

## Credential Priority

### API Tokens

1. **Stored Token** (encrypted in `~/.hyve/credentials.db`)
2. **Environment Variable** (`CIVO_TOKEN`)

Example:
```bash
# 1. Stored token takes precedence
hyve config set-token civo
hyve cluster list  # Uses stored token

# 2. Environment variable used if no stored token
hyve config clear-token civo
export CIVO_TOKEN=your_token
hyve cluster list  # Uses environment variable
```

### Git Credentials

1. **Global Credentials** (encrypted in `~/.hyve/credentials.db`)
2. **Environment Variable** (`HYVE_GIT_TOKEN`)
3. **Per-Repository Username** (if configured during `hyve git add`)

Example:
```bash
# 1. Global credentials (highest priority)
hyve git credentials --username user --password token
hyve git pull  # Uses global credentials

# 2. Environment variable
hyve git credentials --clear
export HYVE_GIT_TOKEN=your_token
hyve git pull  # Uses environment variable

# 3. Per-repository username (lowest priority)
# Set during repository addition with --username flag
```

## Best Practices

### API Token Security

<AccordionGroup>
  <Accordion title="1. Use Interactive Token Entry">
    ```bash
    # Best practice - token not visible
    hyve config set-token civo
    # Prompted for token (hidden input)

    # Avoid - token visible in history
    hyve config set-token civo --token abc123...
    ```
  </Accordion>

  <Accordion title="2. Rotate Tokens Regularly">
    ```bash
    # Generate new token in Civo dashboard
    # Update stored token
    hyve config set-token civo
    # Enter new token

    # Verify
    hyve cluster list
    ```
  </Accordion>

  <Accordion title="3. Use Scoped Tokens">
    When creating tokens in cloud provider dashboards:
    - Limit scope to required permissions
    - Set expiration dates
    - Use separate tokens per environment
    - Document token purpose
  </Accordion>

  <Accordion title="4. Protect Token Files">
    ```bash
    # Ensure proper permissions
    chmod 700 ~/.hyve
    chmod 600 ~/.hyve/credentials.db

    # Never commit credentials
    echo ".hyve/" >> ~/.gitignore
    ```
  </Accordion>

  <Accordion title="5. Clear Unused Tokens">
    ```bash
    # Remove tokens you no longer need
    hyve config list-tokens
    hyve config clear-token old-provider
    ```
  </Accordion>
</AccordionGroup>

### Git Credential Security

<AccordionGroup>
  <Accordion title="1. Use Personal Access Tokens">
    ```bash
    # For GitHub, create PAT with minimal scopes
    # Settings → Developer settings → Personal access tokens

    # Store PAT as password
    hyve git credentials --username your-username --password ghp_...
    ```
  </Accordion>

  <Accordion title="2. Set Token Expiration">
    - Create tokens with expiration dates
    - Rotate before expiration
    - Update Hyve credentials when rotating
  </Accordion>

  <Accordion title="3. Use Environment Variables for CI/CD">
    ```bash
    # In CI/CD pipelines, use environment variables
    export HYVE_GIT_TOKEN=$GITHUB_TOKEN
    hyve git pull
    hyve git push "Automated update"
    ```
  </Accordion>

  <Accordion title="4. Separate Credentials per Environment">
    ```bash
    # Production - stored credentials
    hyve git credentials --username prod-user --password prod_token

    # Development - environment variable
    export HYVE_GIT_TOKEN=$DEV_TOKEN
    ```
  </Accordion>
</AccordionGroup>

## Creating Cloud Provider Tokens

### Civo Cloud API Token

<Steps>
  <Step title="Log into Civo Dashboard">
    Visit [https://dashboard.civo.com](https://dashboard.civo.com)
  </Step>

  <Step title="Navigate to API Tokens">
    - Click on your profile (top right)
    - Select "Security"
    - Click "API Keys" tab
  </Step>

  <Step title="Create New Token">
    - Click "Generate new API key"
    - Name: `hyve-cli` or descriptive name
    - Click "Create"
  </Step>

  <Step title="Copy Token">
    Copy the generated token (shown once)
  </Step>

  <Step title="Store in Hyve">
    ```bash
    hyve config set-token civo
    # Paste token when prompted
    ```
  </Step>
</Steps>

### GitHub Personal Access Token

<Steps>
  <Step title="Log into GitHub">
    Visit [https://github.com](https://github.com)
  </Step>

  <Step title="Navigate to Settings">
    - Click profile picture (top right)
    - Select "Settings"
    - Click "Developer settings" (bottom left)
    - Click "Personal access tokens" → "Tokens (classic)"
  </Step>

  <Step title="Generate New Token">
    - Click "Generate new token" → "Generate new token (classic)"
    - Note: `hyve-git-access` or descriptive name
    - Expiration: Set appropriate expiration
    - Scopes: Select `repo` for full repository access
    - Click "Generate token"
  </Step>

  <Step title="Copy Token">
    Copy the generated token (shown once)
  </Step>

  <Step title="Store in Hyve">
    ```bash
    hyve git credentials --username your-github-username --password ghp_...
    # Paste token as password
    ```
  </Step>
</Steps>

## Environment Variables

### Available Variables

| Variable | Purpose | Example |
|----------|---------|---------|
| `CIVO_TOKEN` | Civo Cloud API token | `export CIVO_TOKEN=abc123...` |
| `HYVE_GIT_TOKEN` | Git authentication token | `export HYVE_GIT_TOKEN=ghp_...` |

### Using Environment Variables

<Tabs>
  <Tab title="Temporary (Current Session)">
    ```bash
    # Set for current terminal session only
    export CIVO_TOKEN=your_token_here
    export HYVE_GIT_TOKEN=your_git_token

    # Use Hyve
    hyve cluster list
    hyve git pull

    # Variables cleared when terminal closes
    ```
  </Tab>

  <Tab title="Persistent (Shell Profile)">
    ```bash
    # Add to ~/.bashrc or ~/.zshrc
    echo 'export CIVO_TOKEN=your_token_here' >> ~/.bashrc
    echo 'export HYVE_GIT_TOKEN=your_git_token' >> ~/.bashrc

    # Reload shell configuration
    source ~/.bashrc

    # Variables available in all new terminals
    ```
  </Tab>

  <Tab title="CI/CD (GitHub Actions)">
    ```yaml
    # .github/workflows/deploy.yml
    jobs:
      deploy:
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v3

          - name: Setup Hyve
            env:
              CIVO_TOKEN: ${{ secrets.CIVO_TOKEN }}
              HYVE_GIT_TOKEN: ${{ secrets.GITHUB_TOKEN }}
            run: |
              hyve git add production --repo-url ${{ github.repository }}
              hyve cluster list
    ```
  </Tab>
</Tabs>

## Credential Migration

### Moving Between Machines

When moving to a new machine:

<Steps>
  <Step title="Export credentials (optional)">
    ```bash
    # On old machine, note your tokens
    hyve config list-tokens
    hyve config get-token civo
    hyve git credentials
    ```
  </Step>

  <Step title="Set up new machine">
    ```bash
    # On new machine, store credentials
    hyve config set-token civo
    hyve git credentials --username user --password token
    ```
  </Step>

  <Step title="Sync repositories">
    ```bash
    # Add repositories
    hyve git add production --repo-url https://github.com/company/hyve-prod.git

    # Sync kubeconfigs
    hyve kubeconfig sync
    ```
  </Step>
</Steps>

<Warning>
  Never copy the `credentials.db` file directly. Encryption keys are machine-specific. Always re-enter credentials on new machines.
</Warning>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Authentication Failed">
    **Problem:** Operations fail with authentication error

    **Solutions:**
    ```bash
    # 1. Verify token is stored
    hyve config list-tokens

    # 2. Check token value
    hyve config get-token civo

    # 3. Re-enter token
    hyve config set-token civo

    # 4. Verify in cloud provider dashboard
    # Token may be expired or revoked

    # 5. Test with environment variable
    export CIVO_TOKEN=your_token
    hyve cluster list
    ```
  </Accordion>

  <Accordion title="Git Operations Fail">
    **Problem:** Git push/pull fails with authentication error

    **Solutions:**
    ```bash
    # 1. Check stored credentials
    hyve git credentials

    # 2. Update credentials
    hyve git credentials --username user --password new_token

    # 3. Verify repository URL
    hyve git list

    # 4. Test with environment variable
    export HYVE_GIT_TOKEN=your_token
    hyve git pull

    # 5. Check token permissions
    # GitHub PAT needs 'repo' scope
    ```
  </Accordion>

  <Accordion title="Token Not Found">
    **Problem:** No token stored for provider

    **Solution:**
    ```bash
    # Store token
    hyve config set-token civo

    # Verify
    hyve config list-tokens
    hyve config get-token civo
    ```
  </Accordion>

  <Accordion title="Credentials Database Corrupted">
    **Problem:** Cannot read credentials database

    **Solutions:**
    ```bash
    # 1. Backup current database
    cp ~/.hyve/credentials.db ~/.hyve/credentials.db.backup

    # 2. Remove corrupted database
    rm ~/.hyve/credentials.db

    # 3. Re-enter credentials
    hyve config set-token civo
    hyve git credentials --username user --password token

    # 4. Verify
    hyve config list-tokens
    hyve git credentials
    ```
  </Accordion>

  <Accordion title="Permission Denied on Database">
    **Problem:** Cannot access credentials database

    **Solutions:**
    ```bash
    # Fix file permissions
    chmod 700 ~/.hyve
    chmod 600 ~/.hyve/credentials.db

    # Verify ownership
    ls -la ~/.hyve/credentials.db

    # Should show your user as owner
    ```
  </Accordion>

  <Accordion title="Token Works in Dashboard but Not Hyve">
    **Problem:** Token works in web UI but fails in Hyve

    **Solutions:**
    ```bash
    # 1. Clear and re-enter token
    hyve config clear-token civo
    hyve config set-token civo

    # 2. Check token scopes/permissions
    # Some operations may require specific permissions

    # 3. Verify token format
    # Ensure no extra spaces or characters

    # 4. Test with curl
    curl -H "Authorization: Bearer YOUR_TOKEN" \
      https://api.civo.com/v2/regions
    ```
  </Accordion>
</AccordionGroup>

## Security Considerations

### What Hyve Encrypts

<CardGroup cols={2}>
  <Card title="API Tokens" icon="key" color="#f43f5e">
    **Encrypted**
    - Cloud provider tokens (Civo, etc.)
    - Stored in `credentials.db`
    - AES-256-GCM encryption
  </Card>

  <Card title="Git Credentials" icon="code-branch" color="#8b5cf6">
    **Encrypted**
    - Username and password/token
    - Stored in `credentials.db`
    - AES-256-GCM encryption
  </Card>

  <Card title="Kubeconfigs" icon="server" color="#3b82f6">
    **Encrypted**
    - Cluster connection details
    - Stored in `kubeconfigs.db`
    - Per-repository encryption
  </Card>

  <Card title="Repository Config" icon="database" color="#6366f1">
    **Not Encrypted**
    - Repository URLs and paths
    - Stored in `repositories.db`
    - No sensitive data
  </Card>
</CardGroup>

### What to Protect

<AccordionGroup>
  <Accordion title="Protect ~/.hyve Directory">
    ```bash
    # Set proper permissions
    chmod 700 ~/.hyve
    chmod 600 ~/.hyve/*.db

    # Never commit to Git
    echo ".hyve/" >> ~/.gitignore

    # Exclude from backups or encrypt backups
    ```
  </Accordion>

  <Accordion title="Protect Shell History">
    ```bash
    # Avoid --token flag usage
    # Bad (visible in history):
    hyve config set-token civo --token abc123...

    # Good (hidden input):
    hyve config set-token civo

    # Clear history entry if token exposed
    history -d <line_number>
    ```
  </Accordion>

  <Accordion title="Protect Environment Variables">
    ```bash
    # Don't echo tokens
    # Bad:
    echo $CIVO_TOKEN

    # Use secure variable storage in CI/CD
    # GitHub Secrets, GitLab CI/CD Variables, etc.
    ```
  </Accordion>

  <Accordion title="Rotate Compromised Credentials">
    If credentials are exposed:

    1. **Immediately** revoke token in provider dashboard
    2. Generate new token
    3. Update Hyve credentials
    4. Update any CI/CD configurations
    5. Review access logs for unauthorized usage
  </Accordion>
</AccordionGroup>

### Audit and Compliance

```bash
# Regular security checks

# 1. Review stored credentials
hyve config list-tokens
hyve git credentials

# 2. Check file permissions
ls -la ~/.hyve/

# 3. Verify token scopes in provider dashboards
# - Civo: https://dashboard.civo.com
# - GitHub: https://github.com/settings/tokens

# 4. Review access logs in provider dashboards

# 5. Rotate tokens on schedule (e.g., every 90 days)
hyve config set-token civo  # New token
hyve git credentials --password new_token
```
